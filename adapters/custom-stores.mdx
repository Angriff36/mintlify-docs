By default, `RuntimeEngine` stores entity state in memory. That works for tests and local development, but production applications need durable storage. Manifest exposes the `Store` interface and a `storeProvider` hook so you can plug in any backend without changing your domain model.

## The Store interface

Every custom store must implement the following interface, exported from `@angriff36/manifest`:

```typescript
interface Store<T extends EntityInstance = EntityInstance> {
  getAll(): Promise<T[]>;
  getById(id: string): Promise<T | undefined>;
  create(data: Partial<T>): Promise<T>;
  update(id: string, data: Partial<T>): Promise<T | undefined>;
  delete(id: string): Promise<boolean>;
  clear(): Promise<void>;
}
```

Each method maps directly to a runtime operation:

| Method | When the runtime calls it |
|--------|--------------------------|
| `getAll` | Hydrating entity collections |
| `getById` | Loading an instance before a command runs |
| `create` | Persisting a new entity instance |
| `update` | Writing state mutations after a command succeeds |
| `delete` | Removing an instance |
| `clear` | Resetting all instances (typically in tests) |

## Wiring in a custom store

Pass a `storeProvider` function in `RuntimeOptions` when you construct `RuntimeEngine`. The function receives the entity name and returns either your store or `undefined` (which falls back to the configured default for that entity).

```typescript
import { RuntimeEngine } from '@angriff36/manifest';
import { MyCustomStore } from './my-custom-store';

const runtime = new RuntimeEngine(ir, {
  userId: 'user-123',
  tenantId: 'tenant-456',
  storeProvider: (entityName) => {
    if (entityName === 'Recipe') {
      return new MyCustomStore({ /* config */ });
    }
    return undefined; // use default memory store for other entities
  }
});
```

## Complete example: Prisma store

Here is a full implementation backed by Prisma. Adapt the delegate type and field mapping to match your schema.

```typescript
import type { Store, EntityInstance } from '@angriff36/manifest';
import type { PrismaClient } from '@prisma/client';

export class PrismaStore<T extends EntityInstance> implements Store<T> {
  constructor(
    private readonly prisma: PrismaClient,
    private readonly model: string
  ) {}

  private get delegate(): any {
    return (this.prisma as any)[this.model];
  }

  async getAll(): Promise<T[]> {
    return this.delegate.findMany();
  }

  async getById(id: string): Promise<T | undefined> {
    const record = await this.delegate.findUnique({ where: { id } });
    return record ?? undefined;
  }

  async create(data: Partial<T>): Promise<T> {
    return this.delegate.create({ data });
  }

  async update(id: string, data: Partial<T>): Promise<T | undefined> {
    try {
      return await this.delegate.update({ where: { id }, data });
    } catch {
      return undefined;
    }
  }

  async delete(id: string): Promise<boolean> {
    try {
      await this.delegate.delete({ where: { id } });
      return true;
    } catch {
      return false;
    }
  }

  async clear(): Promise<void> {
    await this.delegate.deleteMany();
  }
}
```

Wire it in alongside `RuntimeEngine`:

```typescript
import { RuntimeEngine } from '@angriff36/manifest';
import { PrismaClient } from '@prisma/client';
import { PrismaStore } from './prisma-store';

const prisma = new PrismaClient();

const runtime = new RuntimeEngine(ir, {
  userId: session.userId,
  tenantId: session.tenantId,
  storeProvider: (entityName) => {
    switch (entityName) {
      case 'Recipe':   return new PrismaStore(prisma, 'recipe');
      case 'Order':    return new PrismaStore(prisma, 'order');
      default:         return undefined;
    }
  }
});
```

## Built-in storage targets

The runtime ships with first-party support for the following targets:

| Target | Notes |
|--------|-------|
| `memory` | Default. In-process; state is lost on restart. |
| `postgres` | Server-side only. Uses `pg`. Import from `@angriff36/manifest/stores`. |
| `supabase` | Server-side only. Requires the optional `@supabase/supabase-js` peer dependency. Import from `@angriff36/manifest/stores`. |
| `localStorage` | Browser only. |

## Unsupported targets throw — there is no silent fallback

<Warning>
If you pass an unsupported storage target (for example `postgres` in a browser environment), the runtime throws a clear error identifying the unsupported target and entity. It does **not** silently fall back to memory storage.

This behavior is required by the Manifest conformance spec. Always verify your target at startup and surface configuration errors early.
</Warning>

## Practical guidance

- **Keep methods deterministic** in test environments — avoid `Date.now()` or random IDs inside the store; let the runtime supply them via `RuntimeOptions.generateId` and `RuntimeOptions.now`.
- **Enforce tenant boundaries at the storage layer** when your domain requires row-level isolation. The runtime passes `tenantId` in context; your store can filter on it.
- **Surface failures explicitly.** Methods like `update` and `delete` return `undefined` / `false` on not-found, and throw on genuine errors. Do not swallow errors silently.
